# Android release-signing — gradle snippet

> **Task #177 canon.** This is the **only** place where the gradle
> signing block lives. It is **not** committed inside `android/` because
> the entire `android/` directory is gitignored and regenerated from
> `npx cap add android` per machine. After regenerating it, paste the
> snippet below into `android/app/build.gradle` exactly once.

## Why this file exists

Release passwords for the VEX Android app must never live on disk
inside the repo. That rules out:

- `android/app/signing.properties` (even gitignored — written to disk).
- `keystorePassword` / `keystoreAliasPassword` fields in
  `capacitor.config.ts` (Capacitor CLI materialises them into a file).
- `gradle assembleRelease -PstorePassword=…` on the command line
  (visible in the OS process listing).

The chosen pattern is: **gradle reads the four `ANDROID_*` environment
variables directly via `System.getenv(...)` at configure time**. The
build script (`scripts/mobile-android-build.mjs`) validates that the
four env vars are populated, that the keystore file exists, and that
this snippet has been applied (it greps `build.gradle` for
`ANDROID_KEYSTORE_PASSWORD` and refuses to invoke gradle otherwise).

## Where to paste

Open `android/app/build.gradle` after running `npx cap add android &&
npx cap sync android`. Find the `android { ... }` block. Replace any
existing `signingConfigs { ... }` and the matching `buildTypes.release`
with the version below.

## The snippet

```groovy
android {
    // ... existing compileSdk / defaultConfig / etc untouched ...

    signingConfigs {
        release {
            // All four come from the environment. We deliberately do
            // NOT fall back to a properties file or to a default —
            // missing env vars must fail the build, not silently sign
            // with debug or with an empty password.
            def keystorePathEnv = System.getenv("ANDROID_KEYSTORE_PATH")
            def keyAliasEnv     = System.getenv("ANDROID_KEY_ALIAS")
            def storePasswordEnv = System.getenv("ANDROID_KEYSTORE_PASSWORD")
            def keyPasswordEnv   = System.getenv("ANDROID_KEY_PASSWORD")

            if (!keystorePathEnv || !keyAliasEnv || !storePasswordEnv || !keyPasswordEnv) {
                // Empty values trip the assertion in the release
                // buildType below. We intentionally don't throw here
                // (configure phase) because debug builds don't need
                // signing — only release does.
                storeFile     = null
                storePassword = null
                keyAlias      = null
                keyPassword   = null
            } else {
                storeFile     = file(keystorePathEnv)
                storePassword = storePasswordEnv
                keyAlias      = keyAliasEnv
                keyPassword   = keyPasswordEnv
            }
        }
    }

    buildTypes {
        release {
            // Hard fail if signing inputs were missing at configure time.
            doFirst {
                if (signingConfig == null || signingConfig.storeFile == null) {
                    throw new GradleException(
                        "VEX release signing not configured. Export ANDROID_KEYSTORE_PATH, " +
                        "ANDROID_KEY_ALIAS, ANDROID_KEYSTORE_PASSWORD and ANDROID_KEY_PASSWORD " +
                        "before running ./gradlew assembleRelease (see " +
                        "docs/mobile/android-signing-gradle-snippet.md and " +
                        "replit.md § \"Android Release Signing\")."
                    )
                }
            }
            signingConfig signingConfigs.release
            minifyEnabled true
            shrinkResources true
            proguardFiles getDefaultProguardFile('proguard-android-optimize.txt'), 'proguard-rules.pro'
        }
    }
}
```

## How to verify it took

After pasting and saving:

```bash
grep -n ANDROID_KEYSTORE_PASSWORD android/app/build.gradle
```

You should see the four `System.getenv("ANDROID_*")` lines from the
snippet. Then a release build should run end-to-end:

```bash
node scripts/mobile-android-build.mjs bundleRelease
```

If you removed the snippet or copied an old `build.gradle`, the build
script aborts with `android/app/build.gradle does not reference
ANDROID_KEYSTORE_PASSWORD` before invoking gradle.

## Things this snippet deliberately does NOT do

- It does not read `signing.properties`. That file is no longer
  generated by `scripts/mobile-android-build.mjs` and will not be
  recreated by Capacitor (because the corresponding fields were
  removed from `capacitor.config.ts` `android.buildOptions`). If you
  see a `signing.properties` file, delete it — it's stale and unused.
- It does not log the keystore path or any password.
- It does not accept `-PstorePassword=…` on the gradle command line.
- It does not fall back to `~/.gradle/gradle.properties`.
