# قائمة فحص يدوية: سلوك مربع الكتابة فوق لوحة مفاتيح Android (Capacitor)

> هذه القائمة تخدم المهمة #82 وتم تحديثها لإغلاق المهمة #180 (Keyboard config drift). تُنفَّذ على هاتف Android حقيقي (وإن أمكن iOS أيضًا) للتحقق من أن الإصلاح في المهمة #43 (`Keyboard.resize: 'none'` مع `useKeyboardInset` المعتمِد على `visualViewport`) أزال فعلًا فجوة الـ"double-shift" بين مربع الكتابة وأعلى لوحة المفاتيح.
>
> ## ملخّص ما تغيّر في المهمة #180
> - في فترةٍ ما تسرّبت قيمة `Keyboard.resize: 'body'` إلى `capacitor.config.ts`، فصار الـ WebView يعيد تنسيق الجسم في نفس الوقت الذي يحرّك فيه `useKeyboardInset` الـ layout عبر CSS variable. النتيجة: ارتجاج (jitter) ملحوظ على Android كلما فُتحت/أُغلقت اللوحة.
> - التصحيح: إعادة `Keyboard.resize: 'none'` ليبقى الـ JS hook هو المصدر الوحيد للـ inset، مع تعليق صريح يمنع الانحراف مستقبلًا.
> - حماية ضدّ التكرار: ملف اختبار جديد `client/src/hooks/__tests__/keyboard-config-contract.test.ts` يفشل فورًا إذا عاد أحدهم لتعيين `resize` إلى أي شيء غير `'none'`، ويتأكد أيضًا أن الـ hook يحسب الـ inset الصحيح من `visualViewport`.
>
> الملفات المعنية:
> - `capacitor.config.ts` (Keyboard plugin block)
> - `client/src/hooks/use-keyboard-inset.ts`
> - `client/src/hooks/__tests__/keyboard-config-contract.test.ts` (جديد — حارس الانحراف)
> - `client/src/hooks/__tests__/use-keyboard-inset.test.tsx` (موجود — اختبار دورة حياة المستمعين)
> - `client/src/pages/chat.tsx`
> - `client/src/components/games/GameChat.tsx`

## 0) المتطلبات قبل البدء

- [ ] هاتف Android حقيقي (نسخة 11 على الأقل، يفضَّل 13+)، شاشة لمس تعمل، ولوحة مفاتيح Gboard مثبَّتة (اختر Gboard من إعدادات النظام لضمان نتائج موحَّدة).
- [ ] كابل USB + تفعيل وضع المطوّر و"USB Debugging" على الهاتف.
- [ ] على الجهاز المضيف: Node, npm, و Android Studio + Android SDK + Java 17 (أو ما يطابق متطلبات Capacitor الحالية).
- [ ] فرع `main` محدَّث محليًا (`git pull`) ودون تعديلات غير محفوظة.
- [ ] حساب اختبار يحتوي على محادثة DM واحدة على الأقل + غرفة لعبة جارية واحدة (للوصول إلى `<GameChat />`).

## 1) بناء التطبيق وتثبيته

```bash
# من جذر المشروع
npm install
npm run build           # ينتج المجلد client/dist
npx cap sync android    # ينسخ الـ build وإعدادات capacitor.config.ts
```

ثم افتح Android Studio على مجلد `android/`:

- [ ] تأكَّد أن "Run configuration" يستهدف `app`.
- [ ] وصِّل الهاتف عبر USB، اقبل طلب USB Debugging على الهاتف.
- [ ] اضغط Run ▶ — يجب أن يُثبَّت التطبيق ويفتح تلقائيًا.
- [ ] سجِّل الدخول بحساب الاختبار.

## 2) فحص محادثة DM (`/chat`)

افتح صفحة `/chat` ثم محادثة فيها رسائل قديمة (لتظهر أعلى الشاشة):

- [ ] **عند التركيز على مربع الكتابة (المرة الأولى):** لوحة المفاتيح تظهر، ومربع الكتابة يستقر فورًا فوقها مباشرةً. **لا فجوة، لا "قفزة" مزدوجة، لا وميض رمادي**.
- [ ] **اكتب حرفًا واحدًا ثم احذفه:** المسافة بين أعلى لوحة المفاتيح وأسفل مربع الكتابة تبقى صفرًا.
- [ ] **اضغط زر Send:** الرسالة تُرسَل، مربع الكتابة يبقى ملاصقًا للوحة المفاتيح، لا تنغلق لوحة المفاتيح بشكل مفاجئ ثم تُعاد فتحها.
- [ ] **اخرج من المحادثة وارجع إليها:** عند إعادة التركيز يعمل كل شيء بنفس السلاسة (لا يحتاج إعادة تشغيل التطبيق).

## 3) فحص دردشة داخل اللعبة (`<GameChat />`)

ادخل غرفة لعبة جارية وافتح لوحة الدردشة:

- [ ] **التركيز على مربع الكتابة داخل اللعبة:** نفس النتيجة — صفر فجوة، صفر قفزة مزدوجة.
- [ ] **اكتب رسالة بطول سطرين أو ثلاثة (بحيث يكبر مربع الكتابة):** المربع يكبر بسلاسة وكل ما هو أعلاه يتحرك بشكل طبيعي، لا يختفي خلف لوحة المفاتيح.
- [ ] **اضغط خارج المربع لإغلاق لوحة المفاتيح:** لا تظهر فجوة بيضاء لحظية في أسفل الشاشة قبل أن يعود الـ layout لطبيعته.

## 4) اختبار الدوران (Rotation) ولوحة المفاتيح مفتوحة

- [ ] افتح أي دردشة، ركّز على مربع الكتابة، **اللوحة مفتوحة**.
- [ ] أدِر الهاتف من Portrait إلى Landscape — مربع الكتابة يبقى ملاصقًا للوحة المفاتيح بعد اكتمال الدوران، **لا قفزة، لا اختفاء، لا تكدس**.
- [ ] أدِر الهاتف من Landscape إلى Portrait مرة أخرى — نفس النتيجة.
- [ ] كرّر الخطوتين على شاشة DM وعلى `<GameChat />` كليهما.

## 5) شريط الاقتراحات (Predictive Text Bar)

في إعدادات Gboard ← "Text correction"، فعِّل/عطِّل خيار "Show suggestion strip":

- [ ] **مع الاقتراحات مرئية:** افتح الدردشة، ركّز على مربع الكتابة — المربع فوق الاقتراحات مباشرةً (لا فوق اللوحة)، صفر فجوة.
- [ ] **اكتب جزء كلمة:** عند ظهور/اختفاء الاقتراحات لا تتحرك الواجهة بشكل غير متوقع.
- [ ] أعِد الاختبار بعد تعطيل الاقتراحات تمامًا — مربع الكتابة الآن فوق اللوحة مباشرةً، صفر فجوة كذلك.

## 6) (اختياري ولكن مفيد) iOS

إن توفّر iPhone:

```bash
npx cap sync ios
npx cap open ios
```

كرّر الخطوات 2–5 مع مراعاة لوحة QuickType. النتيجة المرجوّة هي نفسها: صفر فجوة، صفر قفزة مزدوجة.

## 7) التوثيق بعد التنفيذ

- إذا اجتاز كل شيء كما هو متوقع: حدِّث `replit.md` بإضافة سطر تحت قسم "Mobile Verification" يقول:
  > Task #82 — Android Capacitor real-device composer behavior verified on `<طراز الهاتف> / Android <الإصدار> / Gboard / <التاريخ>` against the Task #43 fix. No gap, no double-shift, rotation + suggestion-bar toggle clean.
- إذا لاحظت أي سلوك غير متوقع:
  1. سجِّل فيديو قصير للشاشة (Power + Volume Down ثم اختر "Screen record" من شريط الإشعارات).
  2. لاحظ الطراز/إصدار أندرويد/إصدار Gboard وأي إعدادات استثنائية.
  3. ارجع إليّ بالتفاصيل لفتح مهمة متابعة موجَّهة بدلًا من إعادة فتح هذه.
