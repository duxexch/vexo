## 🤖 2. ذكاء البوت — 92/100

### 💪 نقاط القوة (مستوى متقدم جداً):

**تقييم اليد للمزايدة:**
- تقييم دقيق: A(1.5) + K(0.8) + Q(0.4) + J(0.15) أكلة
- مكافأة تسلسل A-K (+0.3) و A-K-Q (+0.5 إجمالي)
- عقوبة K غير محمي (بدون A في نوع قصير): -0.3
- مكافأة الطول (>3 أوراق): +0.4 لكل ورقة إضافية
- مكافأة الفراغ: +0.5 لكل فراغ مع حكم طويل (4+)
- مكافأة المفرد: +0.3
- وزن التوزيع: spread ≥3 يضيف +0.5 (يد غير متوازنة أقوى)
- خصم أنواع جانبية طويلة: -(length-3) × 0.15

**ذكاء المزايدة — نظام متعدد العوامل:**

| الميزة | الوصف |
|--------|-------|
| وعي بالموقع | المقعد الأول: حذر. الثاني: حذر أكثر. الرابع: عدواني إذا الكل مرر |
| مزايدة الشريك | شريك زايد 8+: تعزيز بنسبة تتناسب مع قوتنا (حد أقصى +3) |
| استنتاج خصم | خصم زايد 9+: نقلل بـ1 إذا أضعف منه |
| وعي بالنتيجة | متأخرون: نزيد. متقدمون: ننقص. قرب الهدف: حذر |
| مزايدة تضحوية | الخصم قرب الفوز (85%): نغطي حتى 10 إذا عندنا 5+ أكلات |
| كبوت | 8+ أكلات + فراغ + 12+ تقدير + يكمل الهدف → نزايد 13 |
| مزايدة تنافسية | الخصم يقود: نغطي إذا فرق أكلة واحدة فقط |
| كسر التعادل | الفريقان قرب الهدف: نزايد بجرأة أكبر |

**اختيار الحكم — ذكي:**

| الميزة | الوصف |
|--------|-------|
| نقاط الأونرات | A(+6), K(+4), Q(+2), J(+1) — تقييم شامل |
| مكافأة الطول | (length-3) × 5 للأنواع الطويلة |
| مكافأة الفراغ/المفرد | فراغ(+8), مفرد(+4) في الأنواع الأخرى |
| تعزيز الشريك | شريك زايد 8+ وعندنا 4+ → +bonus |
| عقوبة القصر ضد خصم قوي | خصم زايد 9+ ونوعنا ≤3 → عقوبة |
| عقوبة الثنائي بلا A | نوع من ورقتين بدون A → -3 |

**ذكاء اللعب — نظام متعدد الطبقات (أقوى ميزة):**

| الميزة | الوصف |
|--------|-------|
| ذاكرة أوراق O(1) | `Set` للأوراق الملعوبة + حساب تكرار بالنوع — بحث فوري |
| كشف مسيطر ديناميكي | `isMasterCard()` — الورقة الأعلى المتبقية، ليس فقط الآص |
| تتبع فراغات مباشر | `playerVoids` يُحدَّث لحظياً عند عدم اللزوم (B3) |
| تتبع فراغات استنتاجي | إذا خصم لعب ≤0-1 ورقة في نوع بعد 3+ أكلات → محتمل فارغ |
| إحصاء حكم الخصم | `opponentTrumpsRemaining()` — يخصم حكم الشريك المقدر |
| 4 أوضاع لعب | comfortable / pressured / desperate / aggressiveEndgame |
| مرحلة اللعب ديناميكية | `isEndgame` = أكلات ≥9 أو أكلات مطلوبة ≥ المتبقي-1 |
| سحب الحكم (Bidder) | يسحب أحكام الخصوم بـA/K/Q حتى ينتهوا |
| سحب الحكم (Defender) | يقود حكم إذا خصم فارغ من نوع أو عنده 3+ أحكام قوية |
| قيادة مسيطرة آمنة | يتجنب القيادة بمسيطر في نوع الخصم فارغ منه |
| endgame مسيطرات آمنة | F3-cycle10: يفضل المسيطرات اللي الخصم مش فارغ من نوعها |
| Duck-then-Cash | من A-x-x-x يقود low أولاً ثم يكاش A. من A-K-x-x-x يتبع نفس المنطق |
| إصلاح duck مزدوج | F1-cycle10: A-K-x-x بعد duck لا يعيد duck ثاني |
| تأسيس النوع | 5+ أوراق بدون A → يقود low لسحب أونرات الخصم |
| إدارة المداخل | يقود low في نوع الشريك القوي لإعادة الدور له |
| بطاقة خروج | يحافظ على ورقة في نوع الشريك كمخرج مستقبلي |
| قيادة في فراغ الشريك | إذا الشريك فارغ → يقود من ذلك النوع ليقطع بالحكم |
| كاش مسيطر قبل فراغ | F7-cycle9: يكاش الأونر قبل القيادة Low في فراغ الشريك |
| خلق فراغ استباقي | F10-cycle9: يقود مفرد (غير مسيطر) ليصنع فراغ مع 2+ حكم |
| قيادة عبر القوة | Lead Through: يسار المشتري يقود mid لضغطه |
| قيادة نحو الشريك | Lead Toward: يمين المشتري يقود low لتمرير عبر المشتري |
| ترويج الحكم | يقود J/Q حكم لسحب A/K الخصم وترقية أحكامه |

**ذكاء المتابعة (Following Suit):**

| الميزة | الوصف |
|--------|-------|
| شريك فائز + آخر | يفك الانسداد (Unblock): K-x بلا A → يلعب K |
| شريك فائز + مو آخر | F1-cycle9: يلعب أقل ورقة دائماً (يحافظ على الموارد) |
| المركز الثاني (2nd hand low) | F2-cycle10: يلعب low بدون master، يأخذ بالmaster فقط |
| المركز الثالث (3rd hand) | يغطي بـQ+ لإجبار الرابع على هدر ورقة قوية |
| Finesse كلاسيكي | A-Q ضد K مفقود → يلعب Q |
| Finesse A-Q-J | F7-cycle10: يلعب J أولاً (خسارة أقل لو K ظهر) |
| Finesse K-J | K-J ضد Q مفقود (A خرج) → يلعب J |
| Deep Finesse A-J-10 | A-J-10 ضد K-Q مفقودين → يلعب 10 |
| لا finesse في المركز 4 | F2-cycle8: لا finesse لما نكون آخر يلعب |
| لا finesse ضد ورقة بيدنا | F2-cycle9: لا finesse ضد K إذا K عندنا |
| إشارة العدد | F9: لما ما نقدر نفوز — عدد زوجي=high, فردي=low |
| وضع يائس | يلعب أقوى ورقة لضمان الفوز |
| اقتصاد القوة | يفوز بأقل ورقة فائزة حين ممكن |

**ذكاء القطع (Trumping / Ruffing):**

| الميزة | الوصف |
|--------|-------|
| شريك فائز → لا قطع | يرمي ورقة بدل إهدار حكم |
| قطع بأقل حكم فائز | اقتصاد في المرحلة المبكرة |
| قطع بأعلى حكم | في الوضع اليائس/المتأخر |
| كشف خطر overruff | يفحص كل خصم بعدنا إذا فارغ من النوع وعنده حكم |
| عتبة قطع آمن | F9-cycle9: J في endgame, Q في midgame |
| عتبة overruff متدرجة | F6-cycle10: في المرحلة المبكرة ≤0 لعبات طبيعي |

**ذكاء الرمي (Discarding):**

| الميزة | الوصف |
|--------|-------|
| نظام موحد `pickVoidDiscard` | منطق ذكي لاختيار أفضل ورقة للرمي |
| إشارة سلبية | شريك فائز → يرمي high من نوع ضعيف = "لا تقود هنا" |
| إشارة إيجابية | F6-cycle9: يرمي low من نوع قوي = "قود هنا" |
| خلق فراغ | يرمي من أقصر نوع لصنع فراغ |
| حماية المسيطرات | لا يرمي مسيطر إلا كملاذ أخير |
| بدون حكم = بدون فراغ | F5-cycle10: بدون أوراق حكم، الفراغات عديمة الفائدة → أقل non-master |

**الوضع المريح (Comfortable Mode) — F2:**

| الميزة | الوصف |
|--------|-------|
| كاش مسيطرات آمنة | من أقصر holding أولاً (F7-cycle8) |
| تجنب أنواع الخصم القوية | F12-cycle10: لا يقود في نوع لعب فيه الخصم A/K |
| قيادة low آمنة | يتجنب إعطاء الخصم فرصة قطع |
| fallback طويل | F5-cycle9: أقل ورقة من أطول نوع غير حكم |

**إشارات الشريك:**

| الميزة | الوصف |
|--------|-------|
| قراءة count signal | F4-cycle8: آخر لعبة شريك power ≥8 = count زوجي (أوراق كثيرة) |
| أولوية في الاختيار | F11-cycle10: إذا الأطوال متقاربة (≤1 فرق)، إشارة الشريك تقرر |
| tiebreaker | F4-cycle8: عند تساوي الأطوال، يفضل النوع اللي الشريك يريده |

### ⚠️ نقاط ضعف البوت:

1. **لا يوجد نظام إشارات إرسال واعٍ:** البوت يقرأ إشارات الشريك لكن لا يرسل إشارات منظمة بوعي كامل (الإشارات الحالية ضمنية عبر pickVoidDiscard).
2. **لا يستنتج توزيع أوراق الشريك بالكامل:** يعرف الفراغات فقط + count signal، لكن لا يبني صورة كاملة.
3. **لا squeeze play:** لا يحاول الضغط على خصم يحمل حراسة نوعين مختلفين.
4. **لا يتذكر أخطاء الخصوم:** لا يستغل أنماط لعب خصم ضعيف.
5. **مزايدة الكبوت محافظة:** يزايد 13 فقط عند 8+ أكلات + فراغ + يكمل الهدف — يفوته بعض الكبوتات الممكنة.
