### ⚠️ ملاحظات بسيطة:

1. **مشروع البلوت** يُكتشف فقط في الحكم — وهذا صحيح حسب القواعد التقليدية السعودية

## 🤖 2. ذكاء البوت — 88/100

### 💪 نقاط القوة (مستوى متقدم):

**تقييم اليد للشراء:**
- تقييم أفضل نوع للحكم بناءً على قيم الأوراق + مكافأة الطول
- تقييم الصن مع حساب عدد الآصات وعقوبة الـ10 غير المحمية
- حساب مكافأة المشاريع (تسلسلات + أربع من نوع واحد)
- مكافأة البلوت (K+Q) في الحكم
- مكافأة الفراغ/المفرد للحكم (+12/+6)
- عقوبة فراغ الصن (-8/-4)
- عقوبة الحكم القصير (<3 أوراق = -10)

**ذكاء الشراء:**
- **وعي بالنتيجة:** عتبات متغيرة حسب الفارق في النقاط
- **وعي بتمرير الشريك:** إذا مرر الشريك → يده ضعيفة → رفع العتبة
- **حذر قرب الفوز:** رفع العتبة عند 80% من الهدف لتجنب كبوت كارثي
- **وعي بقرب فوز الخصم:** خفض العتبة للقتال

**ذكاء اللعب — نظام متعدد الطبقات:**

| الميزة | الوصف |
|--------|-------|
| ذاكرة الأوراق | يتذكر كل ورقة لُعبت ومن لعبها |
| بطاقة مسيطرة | كشف ديناميكي للسيطرة (ليس فقط الآصات) |
| تتبع الفراغات | يعرف أي لاعب فارغ من أي نوع |
| قيادة آمنة | يتجنب القيادة بأوراق عالية عندما الخصم فارغ |
| سحب الحكم | يستنزف أحكام الخصوم عبر J/9/A |
| قيادة متسلسلة | يضمن الفوز بفتحات A+K متتالية |
| صيد آخر لفة | يقود بأقوى ورقة مسيطرة لضمان +10 في trick 7 |
| منع الكبوت | يلعب بعنف عند خطر الكبوت |
| صيد كبوت نظيف | ×2 عدوانية عندما الخصم المشتري بـ0 لفات |
| تحليل فجوة النقاط | يائس (<-30) vs مرتاح (>+25) |
| إطعام الشريك | يعطي الشريك أعلى النقاط في آخر موقع |
| تطوير الفراغ | يرمي من أقصر نوع لخلق فراغات |
| تقييم التكلفة | لا يضيع ورقة عالية على لفة رخيصة |
| حفظ المسيطرات | يحفظ الأوراق المسيطرة للقيادة المستقبلية |

### ⚠️ نقاط ضعف البوت:

1. **لا يوجد نظام إشارات (Signaling):** في البلوت الحقيقي، اللاعبون يتواصلون مع شركائهم عبر اختيارات الأوراق. البوت لا يرسل ولا يقرأ إشارات.
2. **لا يستنتج أوراق الشريك:** يتتبع الفراغات فقط، لكن لا يستنتج أوراق الشريك بناءً على طريقة لعبه.
3. **الأرقام السحرية (Magic Numbers):** العتبات مثل 35, 40, 15, -30, +25 مُدمجة مباشرة بدون ثوابت مسماة — تصعب المعايرة.
4. **لا يوجد عد دقيق في النهاية:** لا يحسب النقاط المتبقية بالضبط لمعرفة إذا كان سيفوز أم لا.
5. **لا تمييز في "must-overtake" للبوت:** عند التحكم، يلعب أدنى ورقة فائزة دائماً.

## ⚡ 6. الأداء — 78/100

### 💪 نقاط القوة:

- ترتيب اليد مُخزن مؤقتاً (`useMemo`)
- مراجع (`useRef`) لتتبع التغييرات بدون إعادة رسم
- مؤقت واحد (`setInterval`) وليس عدة مؤقتات
- أدوار البوت تُنفذ متزامنة (synchronous) بدون overhead
- `validMoves` محسوبة على الخادم لتخفيف العبء على العميل

### ⚠️ مخاوف:

1. **`JSON.stringify` / `JSON.parse` على كل حركة:** الحالة الكاملة تُسلسل وتُحلل مع كل حركة.
2. **`structuredClone` على كل حركة:** نسخ عميق للحالة الكاملة — يمكن تحسينه بنظام immutable updates.
3. **لا تحديثات تدريجية:** كل حركة ترسل الحالة الكاملة عبر WebSocket — يمكن تحسينها بإرسال الفروقات فقط (delta updates).
4. **مصفوفة `playedCardsMemo` تنمو:** تتراكم طوال اللعبة (حتى 32 عنصر) — حجم صغير لكن يمكن تحسينه.
5. **60 عنصر `div` للألعاب النارية:** عند الفوز — قد يؤثر على الأجهزة الضعيفة.
